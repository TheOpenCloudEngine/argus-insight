== 개발 환경

=== 소스 코드에서 설치

[source,bash]
----
cd argus-insight-server

# 개발 의존성 포함 설치
make dev

# 서버 실행 (개발 모드, --reload 포함)
make run
----

서버가 `http://0.0.0.0:{server-port}` 에서 시작됩니다.

=== 주요 명령어

[cols="1,2"]
|===
| 명령어 | 설명

| `make dev` | 개발 의존성 포함 설치
| `make run` | 서버 실행 (개발 모드)
| `make test` | 테스트 실행
| `make lint` | 린트 검사
| `make format` | 코드 포맷팅
| `make clean` | 빌드 아티팩트 정리
|===

== 패키지 설치 (rpm/deb)

패키지 설치 시 다음 디렉토리 구조를 사용합니다.

[cols="1,2"]
|===
| 경로 | 용도

| `/etc/argus-insight-server/` | 설정 파일
| `/var/log/argus-insight-server/` | 로그 파일
| `/var/lib/argus-insight-server/` | 데이터 디렉토리
| `/opt/argus-insight-server/` | 애플리케이션 코드
|===

=== systemd 서비스

[source,bash]
----
# 서비스 시작
sudo systemctl start argus-insight-server

# 서비스 상태 확인
sudo systemctl status argus-insight-server

# 부팅 시 자동 시작
sudo systemctl enable argus-insight-server
----

== Swagger UI

서버 시작 후 API 문서를 확인할 수 있습니다.

[cols="1,2"]
|===
| URL | 설명

| `http://localhost:{server-port}/docs` | Swagger UI (대화형 API 테스트)
| `http://localhost:{server-port}/redoc` | ReDoc (읽기 전용 문서)
|===
